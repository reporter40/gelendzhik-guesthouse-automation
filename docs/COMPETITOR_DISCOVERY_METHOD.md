# Методика поиска и отбора конкурентов Акваторинг

**Версия:** 1.0  
**Создан:** 2026-05-14 (Phase C3.5A)  
**Актуальность:** обновлять при добавлении новых объектов или смене рыночной ситуации

---

## 1. Зачем нужна методика

### Проблема случайного отбора

Интуитивный поиск конкурентов приводит к систематическим ошибкам:

- Берётся первая попавшаяся квартира в Геленджике — но у неё 2 спальных места и общая кухня.
- Используется прошлогодняя цена — и система считает рынок дешевле, чем он есть.
- Объект №50 (дом 8 гостей) сравнивают с №40 (2-комнатная квартира 5 гостей) — и рекомендации оказываются неверными.

### Почему нельзя просто «взять несколько цен»

| Ситуация | Последствие |
|----------|-------------|
| Устаревшая цена (>60 дней) | Рынок вырос, мы оставляем старую медиану — теряем доход |
| Неподходящий тип объекта | Медиана занижена, рекомендация на снижение цены |
| Слишком мало источников | Высокая волатильность, ненадёжные рекомендации |
| Объект без кухни посчитан как конкурент | Искажается позиционирование всей когорты |

### Ключевые принципы

1. **Для разных объектов — разные конкурентные корзины.** №50 нельзя сравнивать с обычными 2-комнатными квартирами: у него другая аудитория, другой ценовой сегмент, другой формат.
2. **3 источника — это минимальный seed, не весь рынок.** Начинать с 3+ approved, расширять по возможности до 5–8.
3. **Цель не максимизировать выборку, а получить устойчивую рыночную медиану.** Один очень похожий источник с актуальной ценой ценнее пяти сомнительных.
4. **Confidence cap как защитный механизм.** Пока активных источников < 3, рекомендации для объекта ограничены confidence ≤ 0.40, чтобы система не давала сильных советов при слабой доказательной базе.

## 2. Базовые принципы отбора

| Принцип | Применение |
|---------|-----------|
| Похожее с похожим | Дом 8 гостей → сравниваем с домами 6–8 гостей |
| Качество сигнала важно | Устаревшая цена снижает `signal_quality_score`, но не исключает автоматически |
| Под ключ ≠ посуточная комната | Объект с общей кухней или арендуемыми по отдельности комнатами — не конкурент |
| Премиум и эконом исключаем | Пятизвёздочные апартаменты 25K/ночь или эконом-комнаты 1500/ночь — не наш сегмент |
| Candidate ≠ Active | Пока цена не подтверждена актуально — объект остаётся в `candidate`, не участвует в медиане |
| Территория — маркер сегмента для №50 | Двор/терраса/зона отдыха отличает семейный коттедж от простого этажа |

## 3. Когорты (Competitor Cohorts)

Когорта — это логическая группа объектов, с которыми сравнивается конкретный номер Акваторинга. Один объект принадлежит **одной основной когорте**.

---

### 3.1 `standard_family_2room`

**Целевые объекты Акваторинга:** №40, №41, №42

**Портрет конкурента:**

| Параметр | Диапазон |
|----------|---------|
| Гостей | 4–5 |
| Площадь | 30–55 м² |
| Комнаты | 2 |
| Тип | квартира, этаж под ключ, домик, апартаменты |
| Кухня | обязательно собственная |
| Расположение | Геленджик, Толстый мыс / центр |
| До пляжа | желательно ≤ 700 м |
| Ценовой диапазон | 5 000–7 500 ₽/ночь |

**Признаки соответствия:**
- Объект сдаётся под ключ (а не по комнатам)
- Есть кухня в апартаменте (не общая)
- Подходит семье с детьми 4–5 чел
- Типичная реклама: «2-комнатная квартира», «этаж под ключ 5 чел», «уютные апартаменты»

**Маркеры исключения:**
- Комнаты в гостевом доме (общая кухня, санузел)
- > 6 гостей (другой сегмент)
- Только 1 спальня без гостиной

---

### 3.2 `large_family_house_territory`

**Целевые объекты Акваторинга:** №50

**Портрет конкурента:**

| Параметр | Диапазон |
|----------|---------|
| Гостей | 6–8 |
| Площадь | 50–85 м² |
| Комнаты | 3–4 |
| Тип | дом, этаж под ключ, крупные апартаменты, коттедж, таунхаус |
| Кухня | обязательно собственная |
| Вход | желательно отдельный |
| Территория | желательно двор, терраса, зона отдыха, мангал |
| Расположение | Геленджик, Толстый мыс / центр / ≤ 5 км от моря |
| Ценовой диапазон | 6 500–13 000 ₽/ночь (пик сезона) |
| Формат | семейный отдых, бронирование от 3–7 дней |

**Признаки соответствия:**
- Большой дом или двухэтажный этаж под ключ для семьи
- Мангальная зона, беседка, терраса или закрытый двор
- Собственная кухня с посудой (самостоятельная готовка)
- Типичная реклама: «дом для семьи», «8 гостей», «со своим двором»

**Маркеры исключения:**
- Объект только с 4–5 местами
- Апартаменты в многоэтажном доме без территории
- Гостевой дом с общим двором (конкурент для фонового мониторинга, не для медианы)

> **Важно:** Объект №50 нельзя сравнивать с когортой `standard_family_2room`. Конкуренты из Курзальной 19, Октябрьской и Полевой 29 не должны использоваться как основной рыночный сигнал для №50.

---

### 3.3 `gelendzhik_background_market`

**Целевые объекты:** все (40, 41, 42, 50)

**Назначение:** широкий рыночный фон. Используется только:
- когда основная когорта содержит < 2 active sources
- как дополнительный контекст при сезонном анализе
- для Owner Bot (общая картина рынка в ответах на вопросы)

**Ограничения:**
- Вес сигнала снижен: `background_market_weight = 0.3`
- Не использовать как единственный источник для pricing decision
- Не включает объекты с неподтверждёнными ценами

### Строгие запреты

- **Не обходить капчи** — ни вручную, ни с помощью инструментов
- **Не логиниться на площадки** без явного разрешения (ToS)
- **Не делать агрессивный scraping** — автоматические парсеры, боты, массовый обход
- **Не считать закрытые/недоступные данные надёжными** — если страница требует авторизации, источник недоступен
- **Не копировать фото** конкурентов без надобности

### Оценка надёжности источника

| Источник | Типичный signal_quality | Комментарий |
|----------|------------------------|-------------|
| gelendzhik.travel (верифицирован) | 0.75–0.90 | Актуальные цены, прямые контакты |
| travelandia.ru (актуальные цены) | 0.60–0.80 | Зависит от даты последнего обновления |
| travelandia.ru (цены >1 года) | 0.25–0.45 | Только candidate или background |
| sutochno.ru | 0.55–0.75 | Живые отзывы, цены меняются |
| avito.ru | 0.50–0.70 | Объявления удаляются, нестабильно |
| Ручное наблюдение | 0.80–1.00 | Самое надёжное, если актуально |

## 6. Шкала similarity_score

`similarity_score` — целое число от 0 до 100.

### Для когорты `standard_family_2room` (объекты №40/41/42)

| Критерий | Макс. баллы | Как начисляется |
|----------|------------|-----------------|
| **Расположение / до пляжа** | 20 | ≤300м: 20 / 300–700м: 15 / 700м–1.5км: 8 / >1.5км: 0 |
| **Тип объекта / 2 комнаты** | 20 | 2-комн. квартира/этаж: 20 / 1-комн.+гостиная: 15 / домик 2 комн.: 15 / апарт. без разделения: 8 |
| **Вместимость 4–5 гостей** | 15 | 4–5 гостей: 15 / 3 гостей: 10 / 6: 7 / <3 или >6: 0 |
| **Собственная кухня** | 15 | Кухня в апартаменте: 15 / мини-кухня (без плиты): 8 / общая: 0 |
| **Удобства (AC, WiFi, TV)** | 10 | Все три: 10 / два: 7 / одно: 3 |
| **Семейное позиционирование** | 10 | Явно семейный: 10 / нейтральный: 5 / студенческий/вечеринки: 0 |
| **Ценовой диапазон** | 10 | 5–7.5K: 10 / 4–5K или 7.5–9K: 6 / другое: 2 |

### Для когорты `large_family_house_territory` (объект №50)

| Критерий | Макс. баллы | Как начисляется |
|----------|------------|-----------------|
| **Вместимость 6–8 гостей** | 20 | 6–8 гостей: 20 / 5 гостей: 12 / 9–10: 8 / <5 или >10: 0 |
| **Тип объекта** | 20 | Дом/этаж под ключ/коттедж: 20 / крупные апарт. 60м²+: 15 / таунхаус: 18 / обычная квартира: 0 |
| **Территория/терраса/двор/вход** | 15 | Все (двор+мангал+вход): 15 / двор или терраса: 10 / только балкон: 5 / ничего: 0 |
| **Автономность (кухня+санузел)** | 15 | Полная автономия: 15 / кухня есть, санузел общий: 8 / только кухня: 10 |
| **Расположение / до пляжа** | 10 | ≤500м: 10 / 500м–1.5км: 7 / 1.5–3км: 4 / >3км: 0 |
| **Площадь 50–85 м²** | 10 | 55–85м²: 10 / 45–55 или 85–100м²: 7 / вне диапазона: 3 |
| **Ценовой диапазон** | 10 | 6.5–13K: 10 / 5–6.5K или 13–16K: 5 / другое: 2 |

### Интерпретация similarity_score

| Диапазон | Значение | Рекомендуемый статус |
|----------|----------|---------------------|
| **90–100** | Прямой аналог — почти идентичный конкурент | `approved` / `active` |
| **80–89** | Очень похожий объект, минимальные отличия | `approved` после проверки цены |
| **65–79** | Частичное совпадение — полезен как дополнение | `candidate` |
| **50–64** | Слабое совпадение — только фоновый ориентир | `candidate` / `background` |
| **< 50** | Другой сегмент или формат | `rejected` |

## 7. Шкала signal_quality_score

`signal_quality_score` — число от 0.0 до 1.0 (2 знака после запятой).

Отвечает на вопрос: **насколько можно доверять ценовым данным этого источника?**

### Базовые уровни

| Уровень | Диапазон | Когда применять |
|---------|----------|-----------------|
| **Высокий** | 0.85–1.00 | Свежая цена текущего сезона, явно указана с датой, объект активен, листинг верифицирован |
| **Хороший** | 0.65–0.84 | Цена за текущий или прошлый сезон, объект скорее активен, незначительные пробелы |
| **Средний** | 0.40–0.64 | Цена 1–2 года назад, или «от N рублей» без сезона, или часть параметров оценочна |
| **Слабый** | 0.20–0.39 | Цена 2+ лет назад, или «по договорённости», или нет уверенности в активности |
| **Нет данных** | 0.00–0.19 | Нет цены вообще, объект недоступен, страница удалена |

### Факторы снижения signal_quality

| Фактор | Снижение |
|--------|----------|
| Цена из года N-1 (1 сезон назад) | −0.10 |
| Цена из года N-2 и старше | −0.25 |
| Цена только «от X» без верхней границы | −0.05 |
| Нет информации о сезоне | −0.05 |
| Объект не верифицирован площадкой | −0.05 |
| Нет отзывов на площадке | −0.05 |
| Нет фотографий | −0.10 |
| Страница частично недоступна | −0.15 |
| Противоречивые данные (цена vs. описание) | −0.10 |

### Факторы повышения signal_quality

| Фактор | Повышение |
|--------|----------|
| Листинг явно верифицирован платформой | +0.05 |
| Есть подтверждённые отзывы за текущий год | +0.05 |
| Таблица цен по месяцам с годом | +0.05 |
| Прямой контакт хозяина с возможностью звонка | +0.03 |

## 8. Правила назначения статуса

### `approved` + `status=active`

Условия (все обязательны):
- `similarity_score >= 85` (рекомендуется ≥ 90)
- Цена известна и актуальна (текущий или прошлый сезон)
- Вместимость соответствует когорте
- Тип объекта соответствует когорте
- `signal_quality_score >= 0.60`
- Объект существует и доступен

Результат: участвует в расчёте `market_median`.

---

### `candidate` + `status=candidate`

Условия (хотя бы одно):
- `similarity_score` 65–89
- Цена неактуальна или требует проверки (`signal_quality < 0.60`)
- Объект похож, но не хватает одного ключевого параметра
- Объект новый, ещё не проверен

Результат: **не участвует в market_median**, но виден в UI для ручного решения.

---

### `rejected` + `status=excluded`

Условия (хотя бы одно):
- `similarity_score < 65`
- Общая кухня (не автономный объект)
- Комнаты в гостевом доме
- Эконом < 3 500 ₽/ночь или премиум > 18 000 ₽/ночь без оснований
- Нет кухни в апартаменте
- Не семейный формат (вечеринки, шумные компании)
- Другой регион или удалённость > 10 км от центра
- Для №50: max_guests < 5

Результат: записывается в систему, не участвует ни в чём.

---

### `excluded` (ручной)

Применяется когда объект специально исключён решением команды (например, конкурент переделан в другой формат, закрылся, или сменил сегмент). Отличается от `rejected` тем, что решение принято осознанно, а не автоматически по criteria.

## 9. Специальные правила для объекта №50

### Что нельзя делать

| Запрет | Причина |
|--------|---------|
| Использовать Курзальную 19 / Октябрьскую / Полевую 29 как конкурентов №50 | Это 2-комнатные квартиры 4–5 гостей — совершенно другой сегмент |
| Использовать медиану `standard_family_2room` для №50 | Занизит рынок минимум на 30–50% |
| Одобрять источник с < 6 гостями для когорты `large_family_house_territory` | Искажает медиану вниз |

### Правило confidence_cap

```
IF active_sources_count(large_family_house_territory) < 3:
    confidence_cap = 0.40  ← максимальная уверенность системы ≤ 40%
ELSE:
    confidence_cap = none  ← система работает в полную силу
```

**Текущее состояние (2026-05-14):** `active_sources = 3`, `confidence_cap = снят`.

### Интерпретация рекомендаций для №50

| Сценарий | Что означает |
|----------|--------------|
| `raise_price` + `confidence > 0.5` | Рынок выше, есть основание поднять цену |
| `raise_price` + `confidence ≤ 0.40` | Сигнал слабый (мало данных), поднимать осторожно |
| `market_median` значительно выше текущей цены №50 | Нужна проверка — возможно, мы недооцениваем объект |
| Причина рекомендации ссылается на 2-комнатные квартиры | Баг или устаревшие данные — не применять |

### Приоритет при поиске конкурентов для №50

1. **Обязательно:** собственная кухня
2. **Обязательно:** 6–8 гостей
3. **Желательно:** двор / терраса / территория / мангал
4. **Желательно:** Толстый мыс / центр / до 1.5 км от моря
5. **Опционально:** отдельный вход
6. **Опционально:** 2+ этажа или 3+ комнаты

## 10. Правила обновления данных

### Freshness-шкала для ценовых наблюдений

| Возраст наблюдения | Статус | Вес в расчётах |
|--------------------|--------|---------------|
| **≤ 14 дней** | Свежее | 1.0 (полный вес) |
| **15–30 дней** | Актуальное | 0.8 |
| **31–60 дней** | Стареющее | 0.5 |
| **61–90 дней** | Устаревшее | 0.2 |
| **> 90 дней** | Stale — требует обновления | 0.1 или исключается |

### Регламент обновления

| Период | Частота проверки |
|--------|-----------------|
| **Высокий сезон (июнь–август)** | раз в 7–14 дней для approved источников |
| **Плечо сезона (май, сентябрь)** | раз в 14–21 день |
| **Вне сезона (октябрь–апрель)** | раз в 30–45 дней |
| **Перед принятием решения по цене** | обязательно актуализировать |

### Что делать со stale-данными

1. Найти страницу источника и проверить актуальность
2. Если цена обновилась — добавить новое `competitor_price_observations` с `collection_method='manual_input'`
3. Если объект исчез/закрылся — перевести в `excluded`
4. Если объект сильно изменился — обновить `similarity_score` и пересмотреть статус

### Важные предупреждения

- `seed_from_pdf_report` — это начальная точка, не постоянная истина. Данные из PDF-отчётов устаревают так же, как и любые другие.
- `is_static_price = TRUE` означает: цена не меняется автоматически и требует ручной проверки при каждом использовании.
- Для `candidate`-источников **обязательно подтвердить цену** перед переводом в `approved`.

## 12. Чеклист перед добавлением источника

Перед тем как добавить объект в систему, ответить на все вопросы:

- [ ] Ссылка работает и страница публично доступна?
- [ ] Тип объекта соответствует когорте?
- [ ] Вместимость попадает в диапазон когорты?
- [ ] Есть собственная кухня (не общая)?
- [ ] Цена указана явно или оценочно (указать дату цены)?
- [ ] `signal_quality_score` рассчитан с учётом возраста данных?
- [ ] `similarity_score` рассчитан по весовой таблице раздела 6?
- [ ] Для `approved`: обе оценки ≥ порога (similarity ≥ 85, signal ≥ 0.60)?
- [ ] URL уникален (нет дубликата в системе)?
- [ ] Заполнены `selection_reason` и `data_quality_notes`?
