## Проблема текущего мониторинга

Сейчас мы имеем 3 активных конкурента:
1. Курзальная 19 — similarity 95%
2. Октябрьская — similarity 93%
3. Полевая 29 — similarity 85%

Все три — двухкомнатные квартиры/этажи на 4–6 гостей. Данные получены из PDF-отчёта 2026-05-13, `seed_from_pdf_report`. Ни один не имеет живых актуальных наблюдений.

### Почему 3 источника недостаточно

1. **Нестабильность**: если один источник станет недоступным или owner перестанет обновлять цены — рыночный сигнал деградирует сразу на 33%.
2. **Отсутствие динамики**: мы не знаем, обновляет ли владелец каждого объекта цены активно или цена "заморожена" в начале сезона.
3. **Нет валидации**: медиана из 3 точек легко искажается 1 аутлайером.
4. **Отсутствие peer review**: без cross-check с 7–10 источниками рекомендация даётся с занижением confidence (max 0.75 при 2+ свежих источниках).

### Почему объект №50 нужен отдельный кластер

Объект №50 принципиально отличается от №40/41/42:

| Параметр | №40/41/42 | №50 |
|---------|-----------|-----|
| Тип | квартира/этаж | дом/этаж с территорией |
| Гости | 4–5 | 6–8 |
| Площадь | ~45–55 м² | ~70–85 м² |
| Двор/терраса | нет/балкон | есть |
| Целевая ЦА | семья 4–5, бюджет средний | семья 6–8, бюджет выше среднего |

Если №50 сравнивать с квартирами на 4–5 гостей, мы систематически недооцениваем его рыночную стоимость. Правильный кластер — дома, этажи под ключ, коттеджи 6–8 гостей с двором/территорией.

## Как считаем вес источника

```
weight = (similarity_score / 100)
       × signal_quality_score
       × recency_weight
       × cohort_match_weight
       × static_price_penalty
```

**recency_weight:**
- ≤ 14 дней: 1.0
- 15–30 дней: 0.7
- 31–60 дней: 0.4
- > 60 дней: 0.2

**cohort_match_weight:**
- точное совпадение cohort + apartment_id: 1.0
- background_market: 0.3

**static_price_penalty:**
- `is_static_price = false`: 1.0
- `is_static_price = true`: 0.5

---

## Как оцениваем свежесть

- `last_price_changed_at` — когда в последний раз цена у конкурента менялась
- `price_update_frequency` — ожидаемая частота обновлений: `daily`, `weekly`, `seasonal`, `static`
- `last_price_change_amount` — на сколько изменилась цена (сигнал активности)

Цена считается "живой" если:
- `observed_at < 14 дней` AND `is_static_price = false`

Цена считается "подозрительной" если:
- `observed_at > 30 дней` OR `is_static_price = true`

---

## Как отличаем активный рыночный сигнал от статичной цены

Признаки **активного сигнала**:
- Цена менялась более 2 раз за последние 60 дней
- `price_change_amount != 0` в последних observations
- `signal_quality_score >= 0.7`
- `price_update_frequency` = `daily` / `weekly`

Признаки **статичной цены**:
- Цена не менялась 45+ дней
- `is_static_price = true`
- `signal_quality_score < 0.4`
- `price_update_frequency` = `static` / `seasonal`

## Как это влияет на pricing_recommendations

1. Для №40/41/42 — использовать только `standard_family_2room` источники (priority), фон — `gelendzhik_background_market` с весом 0.3
2. Для №50 — использовать `large_family_house_territory`; при < 3 источниках: `confidence_cap = 0.45`, явная пометка в `reason`
3. При > 60 днях без обновлений у источника — `is_static_price` автоматически ставится, вес снижается
4. Рекомендация с confidence < 0.45 не утверждается автоматически — требует HITL
