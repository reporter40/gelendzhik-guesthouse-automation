## Почему №50 пересчитан отдельно

До Phase C3.5 объект №50 не имел конкурентов в своей категории. Система использовала данные когорты `standard_family_2room` (2-комнатные квартиры, 4–5 гостей) вместо правильной `large_family_house_territory`. В результате:

- `market_median` был занижен: 6 500 ₽ вместо реальных **11 000 ₽**
- Рекомендация предлагала **снизить** цену с 7 000 до 6 300 ₽ — ошибочный сигнал
- `confidence_cap = 0.40` автоматически снижал уверенность из-за отсутствия прямых конкурентов

В Phase C3.5 найдено 9 кандидатов, 3 одобрено. В Phase C3.6 выполнен пересчёт. Теперь №50 сравнивается исключительно с крупными семейными объектами 6–8 гостей с территорией в Геленджике.

## Порядок ручного применения в RealtyCalendar (Phase C2.3 semi-manual export)

Если владелец согласен с рекомендацией:

### 1. Approve в панели /revenue

```
Перейти в /revenue → "Рекомендации по ценам" → Объект №50 → кнопка "Одобрить"
```

Статус изменится: `draft` → `approved`

### 2. Export for RealtyCalendar (Admin API)

Система сформирует `prices_obj` и `manual_instruction`:

```json
POST /admin-api
{
  "action": "pricing_recommendation_export_rc",
  "recommendation_id": "f1f40536-0826-4a06-a0a3-c3f55bbbaeef"
}
```

Ответ содержит:
- `prices_obj` — JSON с новой ценой для RC
- `manual_instruction` — текстовые инструкции для применения в RC вручную
- Статус изменится: `approved` → `exported`

### 3. Применение в RealtyCalendar вручную

По инструкции из `manual_instruction`:
1. Войти в личный кабинет RealtyCalendar
2. Найти объект №50
3. Открыть календарь → 15–17 июля 2026
4. Выставить цену 10 230 ₽/ночь на указанные даты
5. Подтвердить изменение

### 4. Подтверждение в системе

После применения в RC — отметить в панели:
```
/revenue → Рекомендации → №50 → "Отметить как применённое вручную"
```
Статус: `exported` → `manually_applied`
